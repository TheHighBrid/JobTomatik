# JobTomatik v1.00 beta test release

This is the first beta test release for the Android APK.

## What this beta is for

- Verify that Android users can clearly set and test their backend API URL.
- Verify signup and login no longer appear to do nothing when the backend is unreachable.
- Confirm the app shows actionable inline errors instead of relying only on toast messages.

## Before testing the APK

1. Start a JobTomatik backend that the Android device can reach.
2. Open `http://YOUR_BACKEND_URL/health` in a browser and confirm it returns:

   ```json
   {"status":"ok","service":"JobTomatik API"}
   ```

3. In the app, open **API connection** on the login or signup screen.
4. Enter the backend base URL only, for example `http://127.0.0.1:8000` or `http://192.168.1.25:8000`.
5. Do **not** add `/api`; the app adds that automatically.
6. Tap **Test connection** before signing in or creating an account.

## Known limitations

- A backend must be running separately; the APK does not include the backend server.
- On Android, `localhost` works only if the backend runs on the same device.
- Browser automation/Playwright workflows should run on a normal Linux backend or worker, not inside the APK.
